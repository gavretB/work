・Arduino言語（Cベース）のLoRa通信プログラム（2018）
【lora_communication 】
研究活動で自身が作成したプログラムです。今年4月に高知県香美市の市街地でLoRa通信性能測定実験に用いたゲートウェイ用プログラム（test_kami_city/test_lora_gateway）とクライアント（test_kami_city/test_lora_client）それぞれの通信プログラムをアップロードしました。また併せて、昨年1月に作成した戸別ごみ収集システムを想定したゲートウェイ用試作プログラム（prototype_trash_collection/LoRa_Gateway_ver6）とクライアント用試作プログラム（prototype_trash_collection/LoRa_Client_ver6）をアップロードしております。
コーディングにあたって、パケットの送受信に合わせてデータをどのように扱うかを自身で考える必要がありました。不必要な機能やコードは通信性能を下げる要因につながるため、研究の方針を確認し、必要なプログラムを自身で定義した後にコーディングに携わることで、時間的・空間的な無駄やミスの減らす一貫した設計を経験しました。

・Verilog-HDLによるpP（2017・2018）
【hardware_description/pP 】
学部4年次に研究室活動で作成した8bitマイクロプロセッサ：picoProcessorのプログラムです。プロセッサのトップモジュール（pP.v）を各種構成モジュールから構成し、構成モジュールにはプログラムカウンタモジュール（pc.v）、命令解読モジュール（decode.v）、命令メモリモジュール（imem.v）、レジスタモジュール（register.v）、算術論理演算モジュール（alu.v）、データメモリモジュール（dmem.v）、スタックモジュール（stack.v）、割り込みレジスタモジュール（int_reg.v）、2進化１０進数モジュール（bcd.v）、状態コードモジュール（cc.v）、マルチプレクサ各種モジュール（mux1.v, mux2.v, mux3.v）を作成しました。パイプライン処理等のクロックを意識したコーディングを通して回路設計手法再確認することが出来、ハードウェアへの理解をより深めることができました。

・Rubyによる横スクロールアクションゲームプログラム(2018)
【rubycamp2018_spring 】
Ruby発祥の地である島根県で開催されるRuby合宿2018春に参加し、即席チーム＆一週間でRubyを用いた作品をグループ課題として与えられました。大学2年次に異なる言語で横スクロールアクションゲームを作成した経験があったため、メンバーに提案し採用された。5名中、2名はプログラミング経験の浅かったため、副リーダーとしてメンバーをサポートしつつもメンバーの制作物に合わせたステージ構成の提案やアクションの効果音、BGMの導入に携わりました。結果として最優秀賞は得られなかったものの、コンセプトに沿って一つの作品を完成させた経験を積むことができたと同時に、Rubyの魅力を感じることが出来ました。ソースコードについて、(with_pi)はシングルボードRaspberryPiと各種センサーを利用することで稼働可能な合宿当時のプログラムです。(without_pi)は後日、自身でコーディングし直し、キーボードの入力を受け付けるように変更し、RaspberryPiとセンサーなしで稼働可能なプログラムです。

・高知県IoT人材育成コース2018のwebサービス（提案サービス概要資料のみ、ソースコードは未記載）(2018)
【kochi_iot 】
この講座では、前半にIoTとAIについての講義を受講し、終盤のセクションでは5人のグループに分かれて高知県が実際に抱えている問題をIoTやAIを活用することによって解決を図るといったハッカソンを行いました。私の所属するグループでは、高知県の山間部に住む人々が余った野菜を捨てているということに着目し、市場・飲食店までの物流をサポートするサービスを提案しました。具体的には、山村地域の集会所で農家が余った野菜を専用のかごで量りに乗せることで、検知したデバイスがカメラで撮影、webサービスに反映させることで関係者に通知するといったものでした。開発はセンサデバイスサイドとwebサービスサイドに分担して進める方針をとり、わたしはwebサービス側の開発に携わりました。結果として、最優秀賞は得られませんでしたが、このwebサービスではNode.jsを用いて開発を行っており、大学生活を過ごしてきた中でwebサーバはApacheを構築した経験しかなかった私にとって、サーバサイドJavascriptという考え方を学ぶことが出来たことは貴重な経験になりました。

・Verilog-HDLによるgrablu_poker（2017）
【grablu_poker】
学部3年時に研究室活動の夏課題として二人の研究室メンバーと共に作成した回路です。この回路ではモバイルゲーム「グランブルーファンタジー」に登場するポーカーゲームを表現しました。私はポーカーをプレイするのに必要な資金を管理するmoneyモジュールとポーカーの結果を資金へ反映させるmanagementモジュールの作成に携わりました。Verilog-HDLを用いた回路設計はこれが初めてだったため、function分やcase文といったVerilog-HDLの記述法を学びながらコーディングに携わりました。チームメンバーとのスケジュールが合わず、時間が取れなかったためにジョーカーの実装まではいかなかったものの、FPGAボードへの実装し、7セグメントに表示させることでゲームがプレイできる回路を完成させることが出来ました。開発メンバーが理解しやすいコーディング等、チーム開発の難しさを痛感すると同時に、Verilogによる回路設計について理解を深めることができました。
